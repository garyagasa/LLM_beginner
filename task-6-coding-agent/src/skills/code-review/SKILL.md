# Code Review

## 目标

对给定代码变更进行系统化审查，发现问题并提出改进建议。

## 工作流

1. **理解变更**：阅读 diff，理解变更意图
2. **检查正确性**：
   - 逻辑错误
   - 边界条件处理
   - 空值/异常处理
3. **检查安全性**：
   - SQL 注入、XSS、命令注入
   - 敏感信息泄露
   - 权限校验
4. **检查可维护性**：
   - 命名是否清晰
   - 是否有冗余代码
   - 是否需要注释解释
5. **输出审查报告**：列出发现的问题（按严重程度排序）+ 改进建议

## 工具

- `git_diff`：获取代码变更
- `read_file`：读取上下文文件
- `run_tests`：运行相关测试
