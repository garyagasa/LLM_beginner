# Test Runner

## 目标

运行测试套件，对失败用例进行诊断并给出修复建议。

## 工作流

1. **运行测试**：调用 `run_tests` 执行 pytest
2. **解析结果**：
   - 通过/失败/跳过的用例数
   - 每个失败用例的 traceback
3. **诊断失败**：
   - 断言错误：预期值 vs 实际值
   - 导入错误：检查依赖
   - 超时错误：检查性能问题
4. **给出修复建议**：针对每个失败用例建议修复方向
5. **（可选）应用修复**：对确定性高的修复调用 `write_file` + `git_apply`

## 工具

- `run_tests`：执行测试
- `read_file`：读取失败用例的源代码
- `write_file` / `git_apply`：应用修复
